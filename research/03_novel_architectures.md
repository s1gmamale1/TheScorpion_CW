# Novel Architectures 2024–2026
> Beyond the transformer: what's actually deployed vs what's still research

---

## The Big Picture

**2024 was the year alternatives got real.** SSMs, linear attention, and hybrid models moved from "interesting papers" to "deployed models with HuggingFace pages."

**The KV cache is the enemy that drove all of this.** Every alternative architecture's primary sales pitch: "we don't have O(n) growing memory at inference."

**Current equilibrium: hybrids win.** Pure SSMs are faster but lag on in-context recall. Hybrids (SSM + sparse attention) get the best of both.

## Architecture Landscape

### Mamba / Selective SSMs

**Mamba (Dec 2023)** — arXiv:2312.00752
- Creators: Albert Gu (CMU) & Tri Dao (Princeton/Together AI)
- Repo: github.com/state-spaces/mamba

**The selective mechanism:**
```
Classic SSM:  h_t = A·h_{t-1} + B·x_t        (A, B, C fixed for all inputs)
Mamba:        h_t = Ā(Δ_t)·h_{t-1} + B̄(Δ_t, B_t)·x_t   (input-dependent!)
```
B, C, Δ are functions of the current token. Model can selectively remember/forget based on content.

Training: parallel scan (CUDA). Inference: pure recurrence, O(1) per token.

**Mamba2 (May 2024)** — arXiv:2405.21060
- Proves Mamba ≡ structured attention (State Space Duality)
- Multi-head structure → tensor parallelism works
- 2-8× faster training than Mamba1

**Falcon Mamba 7B (TII, 2024):** First serious fully-SSM deployed model at scale.

**Where Mamba struggles:** Exact recall over very long contexts (needle-in-haystack), multi-step reasoning requiring precise lookups. Transformers still win here.

### RWKV — Linear Attention RNN

- Creator: Bo Peng (EleutherAI)
- Repo: github.com/BlinkDL/RWKV-LM
- Paper (v4): arXiv:2305.13048

**Key property:** Dual representation.
- During training: runs as linear transformer (parallelizable)
- During inference: runs as pure RNN (O(1) per token, constant memory)

No attention whatsoever. Best for edge/mobile (constant memory regardless of session length).

**Evolution:**
| Version | Key change |
|---------|-----------|
| RWKV-4 | Time-mixing + channel-mixing, scalar state |
| RWKV-5 (Eagle) | Matrix-valued state, better recall |
| RWKV-6 (Finch) | Data-dependent decay rates — closes gap with Mamba |
| RWKV-7 (Goose, 2025) | More expressive in-context learning |

**Deployed:** RWKV 14B on HuggingFace. Active community at rwkv.com.

### RetNet — Retention Mechanism

- Paper: arXiv:2307.08621 (Microsoft Research)
- Three equivalent representations: parallel (training), recurrent (inference), chunkwise (hybrid)
- Fixed geometric decay (γ^(n-m)) instead of learned content-based weights
- Multi-scale: different heads use different γ decay rates
- **Status: research only.** Not deployed at scale. Influenced later work.

---

### Hyena / Long Convolutions

- Paper: arXiv:2302.10866 (Stanford / Together AI)
- Replaces attention with implicit long convolutions (filter generated by MLP, not stored explicitly)
- O(n log n) via FFT
- Data-independent (unlike Mamba's selective mechanism)

**Deployed:** StripedHyena-Nous-7B (Together AI) — github.com/togethercomputer/stripedhyena

**Hyena DNA:** Applied to genomic sequences (100K+ bp). Outperforms transformers on long-range genomic tasks.

---

### Griffin — Gated Linear Recurrence + Local Attention (Google DeepMind)

- Paper: arXiv:2402.19427
- **RG-LRU (Real-Gated Linear Recurrent Unit):**
  ```
  h_t = α_t ⊙ h_{t-1} + √(1 - α_t²) ⊙ x_t
  ```
  α_t is input-dependent (like Mamba), real-valued, constrained to [0,1]
- Interleaves RG-LRU layers with local sliding-window attention (window=1024)
- **Hawk:** Griffin without any attention — pure recurrence. Still competitive at small scale.

**RecurrentGemma (Google, 2024):** Griffin architecture in production.
- RecurrentGemma-2B and 9B on HuggingFace
- Maps well to TPU matrix units by design
- `huggingface.co/google/recurrentgemma-9b`

---

### xLSTM — Extended LSTM (Hochreiter's Lab)

- Paper: arXiv:2405.04517 (JKU Linz — Sepp Hochreiter, original LSTM inventor)

**Two variants:**
1. **sLSTM:** Exponential gating + normalizer state. Scalar memory like classic LSTM.
2. **mLSTM:** Matrix-valued memory (outer product updates). Fully parallelizable during training — no sequential dependency.

**xLSTM block:** Interleaves sLSTM and mLSTM layers.

**Deployed:** NXAI/xLSTM-7B on HuggingFace (2024).

---

### TTT — Test-Time Training Layers

- Paper: arXiv:2407.04620 (Stanford/UCB/Apple)
- **The hidden state is a model.** At each token, the hidden state is updated via a gradient step on a self-supervised loss.
- Gives the RNN essentially unlimited expressivity in its memory.
- Competitive with Mamba, outperforms transformers on some long-sequence tasks.
- Apple participated → deployment interest hints.
- **Status: research only as of 2025.**

---

### Titans — Long-Term Neural Memory

- Paper: arXiv:2501.00663 (Google DeepMind, 2025)
- Extends TTT with a learned long-term memory module that persists across context
- Three components: short-term attention + long-term neural memory + persistent (parametric) memory
- Strong results on long-context benchmarks
- **Status: research only.**

---

### Liquid Foundation Models (LFM)

- Liquid AI (MIT CSAIL spinoff — Ramin Hasani, Mathias Lechner)
- Based on liquid neural networks / ODE-based recurrences
- LFM-1B, LFM-3B, LFM-7B released 2024
- Claims transformer-matching quality at fraction of inference cost
- **Deployed:** API + HuggingFace. `liquid.ai`

## Where Each Architecture Wins

| Task | Winner |
|------|--------|
| Long sequence throughput + memory | SSM/RNN (Mamba, Griffin, RWKV) |
| Autoregressive inference speed (streaming) | RNN (O(1) per token) |
| In-context learning / multi-step reasoning | Transformer |
| Needle-in-haystack / exact recall | Transformer |
| Genomics / biology (100K+ bp) | Hyena DNA, Evo |
| Edge / mobile deployment | RWKV (constant memory) |
| Best overall quality | Transformer hybrid (still) |

## How to Build Your Own Architecture

### Swappable Components
1. **Attention** — the highest-impact swap. Replace with SSM, linear attention, convolution, or recurrence.
2. **FFN** — swap dense for MoE (sparse activation), or change activation (use SwiGLU).
3. **Normalization** — RMSNorm, pre-norm. Standard now.
4. **Positional encoding** — RoPE for transformers; SSMs don't need it at all.
5. **KV cache strategy** — GQA, MLA, sliding window, or eliminate via SSM.

### Production Viability Checklist
- [ ] Maps to standard matmul ops (tensor cores, no custom CUDA required for baseline)
- [ ] `torch.compile` compatible (no Python control flow that breaks tracing)
- [ ] Tensor parallelism works (attention splits by heads, SSMs need special treatment)
- [ ] Quantizes well (avoid exotic activations with extreme dynamic range)
- [ ] HuggingFace integration (matters enormously for community adoption)
- [ ] Deterministic training stability across seeds

### Hardware-Aware Design

**Apple Silicon:**
- Unified memory = RAM and VRAM are the same pool
- M4 Max: 546 GB/s bandwidth (vs ~50 GB/s for DDR5 PC)
- RNNs (RWKV, Griffin) actually friendlier here than on CUDA GPUs
- MLX framework: github.com/ml-explore/mlx
- MLX-LM for model implementations: github.com/ml-explore/mlx-examples

**Consumer NVIDIA (24GB):**
- Memory-bandwidth bound at batch_size=1 — every design choice should reduce bytes read
- FlashAttention mandatory for context > 8K
- GQA to reduce KV cache, quantization to fit model
- EXL2 for maximum throughput
- Triton for custom kernels: github.com/openai/triton

**The roofline insight:**
```
7B model FP16 = 14GB
RTX 4090 bandwidth = ~1 TB/s
Arithmetic intensity at batch=1 ≈ 2 FLOPS/byte
GPU compute/bandwidth ratio ≈ 82 FLOPS/byte
→ Operating at 2.4% of compute peak. Pure bandwidth problem.
```

## The Field Summary (2026)

1. **Transformer not dead** — but no longer the only serious option
2. **Data quality won** for small models — Phi proved architecture is secondary to data below 10B
3. **Inference efficiency is the new frontier** — MoE, GQA, MLA, SSMs all address this
4. **Apple Silicon created a new deployment target** — unified memory changed what's possible locally
5. **The KV cache is the enemy** — every 2024-2026 innovation is at least partly about eliminating it
